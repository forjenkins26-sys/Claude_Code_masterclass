## Key Lessons Learned

### 1. Facebook Form Structure

- **No semantic attributes:** All inputs have `name="null"`, dynamic IDs
- **Labels are ONLY stable selectors:** Use `getByLabel()` exclusively
- **Dropdowns are comboboxes:** Not native `<select>`, use `role="combobox"`
- **Gender is dropdown:** Not radio buttons as assumed
- **Button text:** "Submit" not "Sign Up"

### 2. Navigation Pattern

Facebook redirects `/reg` to login. Must:
1. Go to main page
2. Click "Create new account" button
3. Wait for form to appear

### 3. Dropdown Interaction Pattern

```typescript
// Click dropdown to open
await dropdown.click();
await page.waitForTimeout(500);

// Select visible option with exact match
const option = page.locator('[role="option"]:visible', { hasText: /^15$/ });
await option.click();
```

### 4. Validation Best Practices

**Don't just check mechanics:**
```typescript
// ❌ BAD
expect(url).toBeTruthy(); // Just checks page loaded
```

**Check actual outcomes:**
```typescript
// ✅ GOOD
expect(page.getByText("What's your first name?")).toBeVisible();
expect(input).toHaveAttribute('aria-invalid', 'true');
```

### 5. Auto-Fix Strategy

1. **Headed mode first** - Don't guess selectors
2. **Fix helper methods, not individual tests** - 1 fix = many tests fixed
3. **Investigate before assuming** - Distinguish test issues from app bugs
4. **Sequential execution** - Prevents Facebook detection/blocking
